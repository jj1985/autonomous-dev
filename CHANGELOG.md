- `pipeline_state.py` `finalize_to_session()` now accepts `feature_ref` and `batch_id` parameters for batch mode (Issue #540). When `feature_ref` is provided, the session record uses schema_version "2.0" and appends per-feature pipeline data to a `features` list, enabling per-feature post-analysis. - `unified_pre_tool.py` Agent Auth layer (Layer 2) now detects `explicitly_invoked` flag and denies Write/Edit/Bash to code files when the coordinator attempts to bypass pipeline agents during an active `/implement` session. Config files and docs remain exempt.

### Fixed
- Terminal-state guard added to `fail_agent()` and `complete_agent()` in `state.py` (Issue #541): both methods are now no-ops when the agent is already in "completed" or "failed" state, preventing a race condition where a late `fail_agent()` call from `unified_session_tracker.py`'s text-scan fallback could overwrite a correctly completed agent.
- `unified_session_tracker.py` status determination now gives priority to `CLAUDE_AGENT_STATUS` environment variable over the `_determine_success()` text scan (Issue #541). The text scan only runs when the env var is absent, eliminating false-failure classifications caused by benign failure-pattern words (e.g. "error handling") in agent output when an authoritative status signal is already present.
- Added `tests/unit/lib/test_agent_tracker_state_idempotency.py` covering terminal-state guard behavior for both `complete_agent()` and `fail_agent()`.
- `unified_session_tracker.py` no longer skips JSONL activity logging when `stop_hook_active=true` is present in the SubagentStop payload (Issue #539). SubagentStop events in consumer repos always carry `stop_hook_active=true`, so the previous early-return guard caused zero activity log entries to be written. The guard has been removed; the hook now logs unconditionally. `_determine_success()` now uses contextual pattern matching to distinguish real failures from benign error mentions (e.g. "error handling", "no errors").
- `RefactorAnalyzer` no longer scans `.worktrees/` or `.claude/` directories, which caused O(n²) timeout on large repos with multiple worktrees (Issue #514). Added `DEFAULT_EXCLUDE_DIRS` class constant (covers version control, caches, worktrees, session logs, archives) and `_should_skip_path()` helper used by all `rglob` traversals. The `__init__` signature now accepts an optional `exclude_dirs` parameter to override the defaults.
